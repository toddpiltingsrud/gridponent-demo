</api>

Events

These are not strictly events, they're delegates. Each event can have multiple delegates. By not using events, we can simplify the function signature of the delegate.

To handle one of these, call the corresponding event with a function:

gridponent('#grid1').beforeInit(function(model){ ... });

  • beforeInit - Called once before the grid is completely initialized.
  • ready - Called after a grid is completely initialized.
  • rowSelected - Called when a table row is clicked.
  • beforeRead - Called before every read operation from the data layer.
  • onRead - Called after a read operation completes.
  • beforeEdit - When an edit operation is about to begin, this is called before showing the edit UI interface.
  • editReady - Called after the edit interface is disaplayed.
  • onEdit - Called after an edit operation completes regardless of whether it succeeded.
  • httpError - Called whenever an http error occurs.

Functions

create(dataItem)

Use this function to create a new record. gridponent will open the record for editing using the specified edit mode.

destroy(dataItem, callback)

Runs the item through the data layer to delete it.

dispose()

Removes all event handlers. This is called by the detachedCallback web component API delegate.

find( selector )

Executes jQuery's find function against the top node of the grid.

getData( uidOrTableRow )

Returns the data row for the specified UID or table row element. If no argument is supplied, returns the entire data array.

getTableRow( dataItem )

Return the table row element that is bound to the data item.

read( requestModel, callback )

Executes the configured read action.

refresh( callback )

Executes the configured read action using the current request model.

search( searchTerm, callback )

Filters the rows displayed in the grid using a basic string comparison algorithm.

sort( fieldName, desc, callback )

Sorts the grid by the specified field name in the order specified by the boolean desc flag.

toggleBusy( isBusy )

Turns the grid's progress indicator on or off according to the specified boolean argument.
